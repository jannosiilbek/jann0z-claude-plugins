# Channel: landing — homepage / landing-page copy

Writes `spec/copy/landing.md`. The page's job is one primary action (the CTA from `product.md` — start
trial / book demo / sign up). Every section earns its place by moving the buyer toward it.

## Section order (drop sections that don't apply; never reorder)

1. **Hero** — Headline + subheadline + primary CTA. The headline carries the positioning statement's
   key benefit in the buyer's words; the subheadline clarifies in 1–2 sentences.
2. **Social proof strip** — logos / a metric / a one-line testimonial. *Only* from positioning proof
   points; if all are `OPEN:`, omit the strip and leave an `OPEN:` marker.
3. **Problem** — name the pain (brief `## Problem`, in customer language). Show you get it.
4. **Solution / benefits** — one block per **messaging pillar** from positioning.md; each = outcome +
   the feature that delivers it (scope reference).
5. **How it works** — 3 steps, reduce perceived effort. Maps to the walking-skeleton flow.
6. **Differentiation / why us** — the wedge + the named-alternative comparison from positioning.
7. **Objection handling** — FAQ or a comparison block, straight from positioning objections.
8. **Final CTA** — restate the core benefit + risk reversal (guarantee/trial from product.md/positioning).

## Headline formulas (offer 2–3, annotate the angle)

- `{Achieve outcome} without {pain}` — e.g. "Publish a covered schedule without blowing the labor budget"
- `The {category} for {audience}` — uses positioning category + persona
- `Never {bad event} again` — leads from the Push force
- `{Pointed question naming the pain}`

## CTA formula

`[Action verb] + [what they get] + [qualifier]` → "Start my free trial", "Get the labor-budget view".
Never "Submit", "Sign up", "Learn more" alone.

## Output structure (in the file)

For each section: the **copy** (ready to paste), then a one-line `— why:` annotation naming the pillar /
objection / force it serves. End with: 2–3 headline alternatives, 2–3 CTA alternatives, and a
`## Meta` block (SEO title ≤60 chars, meta description ≤155 chars) drawn from the same facts.

If a `landing-page` skill is available, hand it this copy to scaffold the page after the file is written.
