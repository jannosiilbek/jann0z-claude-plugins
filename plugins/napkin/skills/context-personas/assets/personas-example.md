# Personas — ShiftLoop

> One block per persona. Each maps to a canonical Role defined in `glossary.md`.
> Permissions live in `rbac-matrix.md`, not here.

## Café Owner

- **Role:** Owner
- **Jobs-to-be-done:**
  - set a weekly labor budget for a Location
  - review labor cost against budget before a Schedule is published
  - purchase and manage the Subscription
  - invite a Manager to a Location
- **Goals:** keep weekly Labor cost within budget without leaving Shifts uncovered.
- **Top pains:** discovers labor overspend only after payroll; cannot see cost while the
  schedule is being built.

## Shift Manager

- **Role:** Manager
- **Jobs-to-be-done:**
  - build a draft Schedule for the upcoming week
  - assign an Employee to a Shift for a required Position
  - publish a Schedule
  - approve or deny a Swap
  - approve or deny a Time-off request
- **Goals:** publish a fully-covered schedule for next week quickly.
- **Top pains:** Swap requests arrive by text and get lost; double-books an Employee by accident.

## Hourly Employee

- **Role:** Employee
- **Jobs-to-be-done:**
  - view assigned Shifts for the week
  - set weekly Availability
  - request a Swap for an assigned Shift
  - request Time-off for a date range
- **Goals:** know when they work and offload a shift they cannot make.
- **Top pains:** finds out about schedule changes too late; no clear way to give a shift away.
